Installation and configuration instructions

Mac OS

Installation:

Installation on Mac is the simplest of all operating systems:

  1. Double-click on the downloaded .dmg file — a window will pop up displaying two options, for example:
    • ESL.bundle
    • Keyboard Installer
  2. Double-click the Keyboard Installer icon, its' window will pop up. At the bottom it says: “Keyboard layout to install: Drag keyboard layout here to install”.
  3. Drag and drop the .bundle icon to the “Keyboard Installer” window, the sentence will change to “Keyboard layout to install: ESL.bundle”.
  4. Click one of the two options:
    • Install for current user (recommended)
    • Install for all users (Sierra or earlier)

Configuration:

  1. Go to:  → System settings → Keyboard.
  2. Find “Text Input” section. In the line “Input Sources” click “Edit” button.
  3. To have a keyboard switcher on the menu bar, turn on the switch next to “Show Input menu in menu bar”
  4. Under “All input sources” (left side) click the “+” icon.
  5. Select language “English” or “Others”.
  6. Select layout to install and click the “Add” button.

Usage:

  • ⌃ Space — press Control + Space to switch layouts - selection window is displayed on the screen.
  • ⌃ ⌥ Space — press Control + Option + Space to switch layouts.

Customizing:

The easiest way to customize layouts please download and install Ukelele from sil.org. Install with administrator privileges — to compile a new .dmg you need to log in as an administrator.

After installation the keyboard layout can be found in either of the two locations:

  1. /Library/Keyboard Layouts/ — all users
  2. ~/Library/Keyboard Layouts/ — current user

Alternatively, you can download layout .bundle file from the project website: github.com/NeilRaiden/keyboard-layouts-macOS/.

To edit the layout, open one of the locations mentioned above and locate the XXX.bundle of the layout you want to edit. It is a directory. However, the Finder will not allow you to get inside. Double-click will open Ukelele.

Alternatively, use the Terminal (or GhosTTY or KiTTY instead). For example:

cd /Library/Keyboard Layouts/ESL.bundle/Contents/Resources/

You should see two files, for example:

  • ESL.icns — a binary file containg the icon set displayed in the status bar.
  • ESL.keylayout — a text file in XML format of the layout. You can edit this file in a text editor.
Windows

There are two options to create a keyboard layout for Windows:

  • Microsoft Keyboard Layout Creator
  • AutoHotKey

As of November 2025, this project offers only scripts created for AutoHotKey v2.

Installation:

  1. Download the keyboard layout of your choice — it is a simple text file (AHK script).
  2. Download and install AutoHotKey version 2
  3. Double-click the .ahk script
  4. Currently (as of November 2025) to disable the layout, you have to stop the script:
    1. Right-click on the AutoHotKey icon in the task bar.
    2. Select “Stop script”.
Linux

Installation:

1. Copy the layout file (for example “esl”) to /etc/xkb/symbols/esl

sudo mkdir -p /etc/xkb/rules
sudo mkdir -p /etc/xkb/symbols
sudo cp esl /etc/xkb/symbols/

2. Add the following section to /etc/xkb/rules/evdev.xml:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE xkbConfigRegistry SYSTEM "xkb.dtd">
<xkbConfigRegistry version="1.1">
  <layoutList>
<!-- ESL (QWERTY) Layout (start) -->
    <layout>
      <configItem>
        <name>esl</name>
        <shortDescription>ESL</shortDescription>
        <description>ESL (QWERTY)</description>
        <countryList><iso3166Id>US</iso3166Id></countryList>
        <languageList><iso639Id>eng</iso639Id></languageList>
      </configItem>
    </layout>
<!-- ESL (QWERTY) Layout (end) -->
  </layoutList>
</xkbConfigRegistry>

3. (optional) add the following line to /etc/xkb/rules/evdev. Note that GNOME layout switcher ignores the evdev file and uses only the evdev.xml:

! option = symbols
  esl:qwerty  = +esl(qwerty)
! include %S/evdev

4. Testing:

  • activate "Writer (QWERTY)": setxkbmap -layout esl,
  • test it by simply typing with the new layout,
  • if setxkbmap returns "Error loading new keyboard description", then it's possible that your Linux distro runs xkb with elevated privileges and ignores files in /etc/xkb/symbols. In such case create a link: sudo ln -s /etc/xkb/symbols/esl /usr/share/X11/xkb/symbols/esl,
  • - reset back to default layout: setxkbmap us

! option = symbols
  esl:qwerty  = +esl(qwerty)
! include %S/evdev
